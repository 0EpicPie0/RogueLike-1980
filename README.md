# Rogue 1980 — Python + curses

Консольная roguelike-игра, воспроизводящая механику оригинального Rogue (1980). Процедурная генерация уровней, туман войны, шесть типов врагов с уникальным ИИ, система предметов, автобаланс сложности и таблица рекордов.

---

## Быстрый старт

```bash
cd src
python3 main.py
```

Требования: Python 3.10+, терминал минимум **80×27** символов, поддержка curses (Linux/macOS).

---

## Управление

| Клавиша             | Действие                 |
| ------------------- | ------------------------ |
| `W A S D` / стрелки | Движение                 |
| `H`                 | Выбрать / сменить оружие |
| `J`                 | Съесть еду               |
| `K`                 | Выпить эликсир           |
| `E`                 | Прочитать свиток         |
| `Q`                 | Сохранить                |
| `ESC`               | Выйти без сохранения     |

## Структура проекта

```
src/
├── main.py                        # Точка входа
├── domain/                        # Игровая логика
│   ├── models.py                  # Все типы данных
│   ├── engine.py                  # Оркестратор: ход игрока, предметы, уровни
│   ├── generator.py               # Процедурная генерация уровней
│   ├── fov.py                     # Ray Casting, туман войны
│   ├── combat.py                  # Формулы боя, атаки
│   ├── ai.py                      # ИИ врагов, патруль, BFS-преследование
│   └── enemy_factory.py           # Фабрика врагов, таблица шаблонов
├── presentation/                  # UI
│   ├── curses_app.py              # Главный цикл, навигация между экранами
│   ├── renderer.py                # Отрисовка карты, HUD, таблицы рекордов
│   └── input_handler.py           # Диалог выбора предмета из инвентаря
└── datalayer/
    └── storage.py                 # JSON-сериализация сессии и рекордов
```

## Автобаланс сложности

| Потери HP | Вывод         | Изменение bias |
| --------- | ------------- | -------------- |
| < 20%     | Слишком легко | +1             |
| 20–50%    | Норма         | 0              |
| 50–70%    | Тяжело        | −1             |
| > 70%     | Очень тяжело  | −2             |

## Типы врагов

| Символ | Враг       | Особенность                                              |
| ------ | ---------- | -------------------------------------------------------- |
| `z`    | Zombie     | Медленный, высокое HP                                    |
| `v`    | Vampire    | Высокая ловкость, первый удар — промах, дренирует max HP |
| `g`    | Ghost      | Невидим до боевого контакта, телепортируется по комнате  |
| `O`    | Ogre       | Огромный урон, ходит на 2 клетки, после атаки отдыхает   |
| `s`    | Snake Mage | Диагональный патруль, 30% шанс усыпить героя             |
| `m`    | Mimic      | Высокая ловкость, высокое HP, низкая сила                |

Враги появляются с глубиной: Zombie и Ghost с уровня 1, Vampire — с 2, Ogre — с 3, Snake Mage — с 4, Mimic - с 5.

---

## Предметы

| Символ | Тип      | Эффект                                     |
| ------ | -------- | ------------------------------------------ |
| `%`    | Food     | Восстанавливает HP                         |
| `!`    | Elixir   | Временно повышает AGI / STR / max HP       |
| `?`    | Scroll   | Постоянно повышает AGI / STR / max HP      |
| `/`    | Weapon   | dagger / mace / axe / spear, бонус к урону |
| `$`    | Treasure | Зачисляется в счёт автоматически           |

## Добавление нового типа врага

1. Добавить значение в `EnemyType` (`models.py`).
2. Добавить шаблон в `_TEMPLATES` (`enemy_factory.py`).
3. Добавить в `enemy_pool_by_depth` при нужной глубине (`enemy_factory.py`).
4. Добавить `case EnemyType.X:` в `_patrol` (`ai.py`).
5. При наличии спецэффекта — добавить в `enemy_attack` (`combat.py`) и `_update_special_state` (`ai.py`).

## Добавление нового предмета

1. Добавить значение в `ItemType` если нужен новый тип (`models.py`).
2. Добавить слот в `Backpack` и ветку в `bucket()` (`models.py`).
3. Добавить весовой коэффициент и генерацию в `_roll_item` (`generator.py`).
4. Добавить обработку в `use_item` → соответствующий `_use_*` метод (`engine.py`).
5. Добавить символ в `_item_symbol` (`renderer.py`).
